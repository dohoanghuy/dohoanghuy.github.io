Structure Programming

By Dijkstra

Programmer can use the hierarchy by the way mathematician do
=> use proven structures => tie them together with code => prove correct themselves

All program can be constructed from just 3 structures:

  • sequence (prove by enumeration)
  • selection (prove by enumeration)
  • iteration (prove by induction with enumeration)

Functional decomposition

Recursively a program to set of small provable function
Program can be proven incorrect by a test, BUT it cannot be proven correct
<=> Software development is not a mathematical endeavor. Rather software is like a science (experiment)

Use a test to try to prove function incorrect
If that test fail => Then we deem the function correct enough for your purpose

Object Oriented Programming

A way to model the real world
3 magic words to explain the nature of OO:

  • encapsulation
  • inheritance
  • polymorphism

Polymorphism is powerful, it’s make dependency inversion, decouple high-level from low-level through interface (plugin architecture)

Function Programming

Immutability - Solution for all the concurrent application with problem:

  • race condition
  • deadlock condition
  • concurrent update

HOW?
Segregation of mutability
Event sourcing
